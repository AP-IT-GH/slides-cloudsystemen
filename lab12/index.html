Laptops dicht
Bind mounts en volumes
  • Beide voor persistente opslag, zoals harde schijf
  • Dus kan container verwijderen, nieuwe container aan oud volume koppelen
  • volumes beheer je via docker volume ... en zijn beter te managen, maar moeilijker te bekijken
  • bind mounts zijn mappen van je lokale machine, gelinkt aan mappen van een container (zoals bij port forwarding)
  • meestal worden volumes aangeraden
  • syntax: -v volumenaam:/map/op/container of -v /lokale/map:/map/op/container
  • toon voorbeeldje uit instructies voor een image op Docker hub
Omgevingsvariabelen
  • niet eigen aan containers, zijn er altijd
  • = variabelen gedefinieerd in bepaalde omgeving
  • vb.: $PATH
  • programma's kunnen deze uitlezen voor configuratie-info
  • mogelijk containers te customizen door deze in te stellen
  • toon voorbeeldje uit instructies voor een image op Docker hub
