Laptops dicht
SSH
  • secure shell
  • applicatielaag
  • vereist client en server
  • client is mee geïnstalleerd met Git bash
  • vaak niet mogelijk via wachtwoord, wel sleutels
  • bijvoorbeeld Github staat geen wachtwoord meer toe
authorized_keys
bijgehouden op de server!
known_hosts
  • bijgehouden op de client
  • adres en signatuur
config
  • meerdere SSH key pairs mogelijk
  • kan default settings koppelen
scp
  • scp /path/to/local/file username@remotehost:/path/to/remote/directory
  • scp username@remotehost:/path/to/remote/file /path/to/local/directory
  • Op Linux begint een filesysteem bij /, niet C:
  • forward slash ipv backslash
Opdracht (Git bash)
  1. sleutelspaar aanmaken: ssh-keygen -t rsa -b 4096 met defaults en lege passphrase
  2. kijk in (verborgen map) .ssh
  3. kopieer naar server: ssh-copy-id -i ~/.ssh/id_rsa.pub username@remotehost
    • dit combineert scp met een append
  4. log in op de server
  5. chmod 700 ~/.ssh beperkt rechten map
  6. chmod 600 ~/.ssh/authorized_keys beperkt rechten file
  7. test login zonder wachtwoord
  8. registreer public key in Github
  9. maak een niet-lege repo, clone via SSH, commit iets, push
