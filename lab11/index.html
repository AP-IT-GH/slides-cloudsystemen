Laptops dicht
container vs. VM
  • VM = van de grond op (simulatie HW)
  • container = enkel exact wat applicatie nodig heeft
    • tweede VM = dubbele kost
    • tweede container = minimale extra kost
  • In de eerste plaats Linux, omdat images dus die kernel delen.
image vs. container
  • "template" vs. instantie
  • vergelijk met "klasse" vs. "object"
  • filmpje: image = onderste lagen (liggen vast), container heeft jouw eigen extra bovenste laag
Klassikale workshop (deel 1)
Doorlopen "Build a containerized..." op Azure tot en met "Customize a Docker image to run your own web app."
Opdrachten
  • Maak een nieuwe Dockerfile.
  • Zorg dat hij voortbouwt op ubuntu:latest
  • Zorg dat hij beschikt over curl. In Ubuntu installeer je dit via apt-get update && apt-get install -y curl.
  • Zorg dat hij de broncode van neverssl.com toont in de terminal wanneer hij start.
  • Test uit door hem te builden en uit te voeren.
  • Download "voorbeeldapplicatie Express" (onder "Applicatielaag: HTTP")
  • Maak een nieuwe Dockerfile.
  • Zorg dat hij voortbouwt op ubuntu:latest
  • Zorg dat hij een map /app heeft.
  • Kopieer de bestanden hello.js, package.json en package-lock.json naar deze map.
  • Zorg dat de nodige packages geïnstalleerd worden en dat de applicatie wordt opgestart wanneer de container start.
  • Zorg ook dat hij naar de gebruikte poort luistert.
  • Build en start de container. Gebruik `-p HOSTPOORT:GUESTPOORT` (met daar de juiste poortnummers) om te port forwarden.

Workshop (deel 2)
Doorlopen. Niet overal hetzelfde als in schriftelijke uitleg ⇒ vraag het aan de lector! Wissen resource group op het einde: moet specifiek op detailpagina van die groep staan.
  • Stop je container met de Express applicatie.
  • Maak een Docker hub account aan. Gebruik best geen temp mail.
  • Registreer de image van de vorige opdracht als publieke repository (dat is gratis).
  • Vraag een buur om zijn/haar namespace en repository name.
  • Run de image van je buur, opnieuw met de juiste port forwarding.
  • Kan je de applicatie bezoeken?
